Netzwek Chat-Projekt

Webbasierte Präsentation zur Chat-Anwendung mit C-Server und browserseitiger Verschlüsselung.

Benutze die Pfeile unten, um die Folien zu wechseln.

Projektübersicht

  • Server-seitige Anwendung in C mit Winsock.
  • Client-seitige Chat-Oberfläche in HTML/CSS/JavaScript.
  • Nachrichten werden per HTTP an den Server gesendet und als JSON zurückgegeben.

Architektur

  1. Browser lädt Site.html.
  2. js.js sendet verschlüsselte Nachrichten an /api/send.
  3. Server speichert die Nachricht unverändert.
  4. Browser fragt /api/messages ab und entschlüsselt die Nachrichten.

Verschlüsselung

Die Nachricht wird mit einem einfachen Caesar-Verfahren verschlüsselt.

Der Schlüssel ist die Länge des Benutzernamens:

shift = username.length

Beispiel:

username = "Du" → shift = 2
message = "abcdefg" → gesendet: "cdefghi"

Serverrolle

  • Empfängt HTTP-POST an /api/send.
  • Parst username und message.
  • Speichert die verschlüsselte Nachricht in einem Array.
  • Antwortet auf /api/messages mit allen gespeicherten Nachrichten.

Dateien

  • server.c – TCP-Server in C.
  • Site.html – Chat-Interface.
  • js.js – Weblogik, Verschlüsselung und Anzeige.
  • style.css – CSS für das Chat-Interface.
  • security.js – zusätzliche Verschlüsselungsfunktion, aber nicht benötigt.

Build & Nutzung

Server kompilieren mit:

gcc -o server.exe server.c -lws2_32

Dann server.exe starten und im Browser Site.html öffnen.

Wichtige Hinweise

  • Die Verschlüsselung ist einfach und dient nur als Demo.
  • Im Netzwerk sieht man die verschlüsselte Nachricht im HTTP-Body.
  • Für echte Sicherheit müsste man HTTPS und einen stärkeren Algorithmus verwenden.

Zusammenfassung

Dieses Projekt zeigt eine einfache Chat-Architektur mit:

  • Client-seitiger Verschlüsselung
  • Server-Speicherung verschlüsselter Nachrichten
  • JSON-basierter Kommunikation
  • Webbasiertem Chat-Interface
